## Technical Notes

### KML to GeoJSON Conversion
- KML coordinates format: `lon,lat,alt lon,lat,alt`
- GeoJSON format: `[[lon, lat], [lon, lat]]`
- Parser automatically closes polygons (first = last coordinate)
- Area calculation uses spherical geometry (accurate for lat/lon)

### Database Relationships
- `field_zones` (1) → (many) `health_indices`
- `field_zones` (1) → (many) `alerts`
- `satellite_images` (1) → (many) `health_indices`
- Cascading deletes configured for data integrity

### API Design
- FastAPI automatic OpenAPI documentation at `/docs`
- Database sessions managed via dependency injection
- CORS enabled for future dashboard access
- All decimal values converted to float for JSON serialization
