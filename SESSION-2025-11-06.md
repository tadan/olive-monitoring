## Technical Architecture

### Data Flow

```
1. QUERY
   ↓
[Copernicus API] → List of Sentinel-2 products
   ↓
2. DOWNLOAD
   ↓
[Sentinel-2 .zip] → Extract → [.SAFE directory]
   ↓
3. PROCESS
   ↓
[Band Files] → Clip to zones → [Red, NIR, SWIR arrays]
   ↓
4. CALCULATE
   ↓
[NDVI, NDMI] → Statistics → [Health Score]
   ↓
5. STORE
   ↓
[Database] → SatelliteImage + HealthIndex records
   ↓
6. ANALYZE
   ↓
[Alert Detector] → Compare with baselines → [Alerts]
   ↓
7. NOTIFY
   ↓
[Email] → HTML formatted alerts → [Farmer]
```

### Database Updates

Phase 2 will populate these tables:
- `satellite_images` - Metadata for each Sentinel-2 product
- `health_indices` - NDVI/NDMI/health score time-series
- `alerts` - Detected anomalies and issues
- `baseline_statistics` - Seasonal baselines for anomaly detection

## Key Design Decisions

### 1. Sentinel-2 Band Selection
- **Red (B04):** 10m resolution, 665nm wavelength
- **NIR (B08):** 10m resolution, 842nm wavelength
- **SWIR (B11):** 20m resolution, 1614nm wavelength

Rationale: These bands provide optimal NDVI (vegetation vigor) and NDMI (moisture) calculation for olive groves.

### 2. Cloud Coverage Threshold
- Default: 30% maximum
- Testing: 50% maximum (relaxed)
- Configurable via command-line

Rationale: Balance between data availability and quality.

### 3. Alert Thresholds

**NDVI Drop:**
- Warning: 15% decrease vs previous measurement
- Critical: 25% decrease

**Drought Stress (NDMI):**
- Warning: NDMI < -0.2 (moderate stress)
- Critical: NDMI < -0.3 (severe stress)

**Waterlogging:**
- Warning: NDMI > 0.5 (high moisture)
- Critical: NDMI > 0.6 (excessive moisture)

**Health Score:**
- Warning: Score < 60
- Critical: Score < 40

Rationale: Based on agronomic research for Mediterranean olive cultivation.

### 4. Baseline Requirements
- Minimum 3 samples per baseline
- Seasonal baselines (spring/summer/fall/winter)
- Overall baseline (all seasons) as fallback
- 2σ threshold for anomaly detection

Rationale: Statistical rigor while handling limited historical data.

### 5. Processing Schedule
- Recommended: Every 5 days (matches Sentinel-2 revisit)
- Search period: Last 7 days
- Initial run: 30 days back to build baselines

Rationale: Balance between timeliness and computational load.

## Known Limitations & Future Enhancements

### Current Limitations

1. **Single Zone Query:** Currently queries products using first zone's geometry. Could be enhanced to use union of all zones for better coverage.

2. **No Cloud Masking:** Image processor clips to zone boundaries but doesn't mask cloudy pixels within zones. Future enhancement: use SCL (Scene Classification Layer) band.

3. **Sequential Processing:** Zones are processed sequentially. Could be parallelized for better performance.

4. **Fixed Thresholds:** Alert thresholds are hardcoded. Could be made configurable per zone or crop type.

5. **No Retry Logic:** Download failures aren't retried. Could add exponential backoff retry mechanism.

### Future Enhancements

1. **Advanced Analytics:**
