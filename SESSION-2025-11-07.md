## Issues Encountered ⚠️

### Critical Issue: Satellite Tile Mismatch

**Problem:**
```
ValueError: Input shapes do not overlap raster
Intersection is empty Window(col_off=-39995, row_off=469999, width=1, height=1)
```

**Root Cause:**
- Farm coordinates: 42.303°N, 14.187°E (Abruzzo, Italy)
- Downloaded tile: T33TVG
- Farm and satellite tile don't overlap

**Why This Happened:**
- Copernicus query uses zone polygon to find intersecting products
- Query returns tiles that *partially* intersect search area
- But farm itself isn't in those tiles (adjacent tiles returned)

**Attempts to Fix:**
1. ✅ Added CRS transformation (EPSG:4326 → UTM)
2. ✅ Filtered to L2A products only
3. ❌ Still fails - wrong tile downloaded

**Next Steps to Resolve:**
1. Identify exact MGRS tile containing farm (likely 33TUG or 33TTG, not 33TVG)
2. Add tile filter to satellite query
3. Or use point geometry (farm center) instead of polygon for initial query
4. Re-run processing with correct tile

```

3. Use Bearer token for API requests
4. Auto-refresh when expired
```

**Product Query Flow:**
```
1. Transform zone geometry to WKT
2. Build OData filter with:
   - Collection: SENTINEL-2
   - Geometry intersection
   - Date range
   - Cloud coverage < threshold
   - Product type: S2MSI2A (L2A only)
3. Query catalogue.dataspace.copernicus.eu/odata/v1
4. Parse results and extract metadata
```

## Key Decisions Made

1. **OAuth2 Implementation:** Replaced sentinelsat library entirely with direct HTTP calls
   - Rationale: Library not compatible with new Copernicus Data Space
   - Benefit: Full control over authentication and queries

2. **L2A Products Only:** Filter to exclude L1C products
   - Rationale: L1C has different folder structure, causes band errors
   - Benefit: Consistent processing path, atmospherically corrected data

3. **CRS Transformation:** Transform coordinates before clipping
   - Rationale: Zone coordinates in lat/lon, raster in UTM
   - Benefit: Proper geospatial alignment (when correct tile available)

4. **Task Scheduler vs Cron:** Used Synology Task Scheduler GUI
   - Rationale: Better integration with DSM, email notifications
   - Benefit: Easier management, survives updates

5. **Stop Processing:** Decided to stop and debug rather than continue failing
   - Rationale: Processing wrong tiles wastes bandwidth and disk space
   - Benefit: Clean restart once tile issue resolved
